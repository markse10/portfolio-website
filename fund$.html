FUND$

Anyone can save money!

Project Overview

FUND$ is a financial app for people who don't like working with their finances but want to save money quickly for a purpose, i.e. vacation, wedding, down payment, etc. FUND$ will appeal to users that are less familiar with their finances and will provide simple, clear messaging as well as offer guidance and motivation for users to make savings fun.

A finance app that you can control, gain personal insights, and receive the motivation you need to save for the special moments in your life.

Brand Values

  • Trust
  • Reliability
  • Simplicity
  • Motivational
  • Accessible

Define your user

Before designing my app, I needed to learn about what users want in a money-saving app. I needed to define the 5 W's, define my user stories, and create a user flow.

  • Who?
  • What?
  • When?
  • Where?
  • Why?

User Stories

Who is my user and what do they want to do with this app?.

Track Spending

As a user, I want to see a dashboard of my finances clearly so that I can see how much I am spending on what at a glance.

Set Goals

As a user, I need to be able to tell the tool what my savings goal is and how long I have to reach it, so I can save accordingly.

Create a Budget

As a money-saver, I need to be able to input information on the money I am receiving and spending (and on what), so that I can see an overview of my finances.

Low-Fidelity Wireframes

At this point in my project, I started designing the initial app screens as low-fidelity wireframes

Brand Logo

When you think of saving money, you think of a piggy bank. I wanted to evoke a sense of trust and simplicity with the FUND$ app. Using a simple piggy bank outline with a coin is instantly recognized as a symbol of saving money.

Brand Colors

The color scheme I chose provokes a sense of trust while remaining cool and calm. The yellow accent is there to show happiness, joy, and accessibility. FUND$ strives to evoke feelings of genuinity, positivity, and caring.

Brand Typography

FUND$ strives for clear and concise messaging. We want our brand to be bold, simple, and easy to read so I elected to use the Roboto font.

Mid-Fidelity Wireframes

After coming up with the style guide for FUND$, it was time for me to start on my Mid-Fidelity Wireframes so I could further develop my design and start user testing.

User Testing

In the user testing phase, I wanted to pick certain scenarios that were needed for basic use of the app.

Scenario 1

From the dashboard, navigate to your transactions and back to dashboard. Then navigate to see your savings goals.

Scenario 2

Open a savings goal and add money to it.

Scenario 3

Sort transactions, click on them to edit the category if needed.

Revisions

After completed my user testing, I recorded some user responses and made the appropriate changes.

User 1

"I don't like the graph icon. I would want to see an actual graph of my current financial status."

I will add an actual Sankey graph diagram with real categories and totals.

User 2

"It could be confusing to see what is a debit and credit on the transactions page. Are you going to make sure they stand out?"

When designing this for my final screens, I will make sure debits and credits are different colors so there is no confusion.

User 3

“Change the add money bar to fund gift or something. I wasn’t sure what I was adding money to. On the add money screen, there is a save function. Is that actually funding the goal or just a saved intention?”

I changed the name of the screen to Fund Goal and changed the saved button to a Fund button.

User 4

“There is not a due date on the goal information page. It would be nice to also see how much you have saved towards your goal on the details page.”

I will add a due date on the Fund Goal screen. I will also include the amount saved towards the target amount on the details page.

Key Takeaways

FUND$ was the latest project I completed for my CareerFoundry program. I have definitely learned so much and I think it definitely shows in my work. I really loved the branding stage of this project. I definitely see the need to develop more screens and I would like to work on the notifications and locked screen messaging. All and all I learned a lot and I am excited to continue gaining new skills and developing into a stronger UI/UX designer. I also learned that I love to create the case study for a project. I love how creative you can be and how all your work really comes together.
